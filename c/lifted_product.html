Lifted-product code

Name
Lifted-product code[1]
Description
A QLDPC code that is the "lifted version" of a hypergraph product code. The construction of a lifted product code is similar to that of a hypergraph product code; a sort of ''product'' is taken between two input classical codes to produce a CSS code. What differentiates a lifted product code from a hypergraph product code is that a reduction of symmetry is exploited to decrease the number of physical qubits required. Lifted product codes are notable because they were the first QLDPC codes to achieve almost linear distance.
Protection
Code performance strongly depends on \(G\) [2], but lifted product codes contain [3] families of QLDPC codes with the parameters \([[n, k = \Theta(n), d = \Theta(n)]]\).
Rate
Lifted-product codes include the first examples [3] of (asymptotically) good QLDPC codes, i.e., codes with asymptotically constant rate and linear distance. The existence of such codes proves the QLDPC conjecture [4]. Another notable family encodes \(k \in \Theta(n^\alpha \log n)\) logical qubits with distance \(d \in \Omega(n^{1 - \alpha} / \log n)\) for any number of physical qubits \(n\) and any real parameter \(0 \leq \alpha < 1\) [2]. There is no known simple way to compute the logical dimension \(k\) in the general case [2].
Decoder
BP-OSD decoder [1] adds OSD-like post-processing to the belief propogation decoder for improved performance.
Notes
Formerly known as generalized hypergraph product codes [1], and later renamed to lifted-product codes [2][4].
Parent(s)
  • Homological product code (Lifted product obtained from the homological product over a group algebra \(\mathbb{F}_2 G\) of two length-two chain complexes over \(\mathbb{F}_2 G\), i.e., classical binary codes with a free action of the group \(G\).)
  • Balanced product code
Parent of
  • Hypergraph product code (Lifted-product codes for trivial group \(G\) are hypergraph-product codes.)
