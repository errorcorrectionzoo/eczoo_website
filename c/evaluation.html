Evaluation AG code

Description

Evaluation code on a set of points \({\cal P} = \left( P_1,P_2,\cdots,P_n \right)\) lying on an algebraic curve whose corresponding vector space of functions \(f\) is a Riemann-Roch space \(L\) ([1], pg. 313), i.e., a vector space of rational functions with certain behavior of the functions's zeroes and poles. Being an evaluation code, its codewords are evaluations of all functions at the specified points, \begin{align} \left( f(P_1), f(P_2), \cdots, f(P_n) \right) \quad\quad\forall f\in L~. \end{align}

Protection

Riemann-Roch theorem yields code length \(n\), dimension \(k\), and a lower bound on distance in terms of features of \(L\) ([1], Thm. 15.3.12). The order or Feng-Rao bound, a generalization of the shift bound for cyclic codes, gives a lower bound on the distance of evaluation AG codes [2][3][4].

Parents

  • Algebraic-geometry (AG) code
  • Evaluation code — Evaluation AG codes are evaluation codes for which \(\cal X\) is an algebraic curve, i.e., an algebraic variety of dimension one [5].
  • Projective code — An AG-code generator matrix has columns that form a subset of points on a curve in projective space.

Children

  • Elliptic code — Elliptic codes are evaluation AG codes with \(\cal X\) being an elliptic curve, i.e., curve of genus one ([6], Ch. 3.2; [5]).
  • Generalized RM (GRM) code — GRM codes are evaluation codes of multivariate polynomials for which \(\cal X\) is an \(m\)-dimensional affine space over \(GF(q)\) ([6], pgs. 44-46; [5]).
  • Generalized RS (GRS) code — GRS codes are in one-to-one correspondence with evaluation AG codes of univariate polynomials \(f\) with \(\cal X\) being the projective line ([1], Thm. 15.3.24; [6], Ch. 3.2; [5]). RS codes are evaluation codes on the affine line.
  • Hermitian code — Hermitian codes are evaluation AG codes with \(\cal X\) being a Hermitian curve ([5], Ex. 2.74).
  • Hexacode — The hexacode is an evaluation AG code over \(GF(4) = \{0,1,\omega, \bar{\omega}\}\) with the curve \(\cal X\) defined by the equation \(x^2 y + \omega y^2 z + \bar{\omega} z^2 x = 0\) ([5], Ex. 2.77).
  • Klein-quartic code — Klein-quartic codes are evaluation AG codes with \(\cal X\) being the Klein quartic ([5], Ex. 2.75).
  • Plane-curve code — Plane-curve codes are evaluation AG codes with \(\cal X\) being the plane curve, which is defined using a bivariate polynomial ([5], Thm. 2.27).
  • Reed-Muller (RM) code — RM codes are evaluation codes of multivariate polynomials for which \(\cal X\) is an \(m\)-dimensional affine binary space ([6], pgs. 44-46; [5]).
  • Residue AG code — Any residue code can be equivalently stated as an evaluation code ([1], Lemma 15.3.10; [5], Thm. 2.72). In addition, evaluation and residue AG codes are dual to each other ([1], pg. 313; [5]).
