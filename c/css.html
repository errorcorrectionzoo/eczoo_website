Calderbank-Shor-Steane (CSS) stabilizer code

Name
Calderbank-Shor-Steane (CSS) stabilizer code[1][2][3]
Description

An \([[n,k,d]]\) stabilizer code admitting a set of stabilizer generators that are either \(Z\)-type or \(X\)-type Pauli strings. The parity check matrix is of the form \begin{align} H=\begin{pmatrix}0 & H_{Z}\\ H_{X} & 0 \end{pmatrix}~. \label{eq:parity} \end{align}

Encoding is based on two binary linear codes, an \([n,k_X,d_X]\) code \(C_X\) and \([n,k_Z,d_Z]\) code \(C_Z\) with \(C_X^\perp \subseteq C_Z\), such that \(k=k_X+k_Z-n\) and \(d\geq\min\{d_X,d_Z\}\). The \(H_X\) (\(H_Z\)) block of \(H\) \eqref{eq:parity} is associated with the code \(C_X\) (\(C_Z\)), and the requirement \(C_X^\perp \subseteq C_Z\) guarantees that the \(X\)-stabilizers of \(C_X\) commute with the \(Z\)-stabilizers of \(C_Z\).

A CSS code has stabilizer weight \(w\) if the highest weight of any stabilizer generator is \(w\), i.e., any row and column of \(H_X\) and \(H_Z\) has weight at most \(w\). In the context of comparing weight as well as of noise models biased toward \(X\)- or \(Z\)-type errors, an extended notation for CSS codes is \([[n,k,(d_X,d_Z),w]]\).

Protection
Detects errors on \(d-1\) qubits, corrects errors on \(\left\lfloor (d-1)/2 \right\rfloor\) qubits.
Encoder
Stabilizer measurement [4].
Notes
Original requirement of \(C_X^\perp \subset C_Z\) [1] has been relaxed to absorb hypergraph product codes.
Parent(s)
  • Stabilizer code (Stabilizer generators can be expressed as either only \(X\)-type or only \(Z\)-type. However, any \([[n,k,d]]\) stabilizer code can be mapped onto a \([[4n,2k,2d]]\) weakly self-dual CSS code, with the mapping preserving geometric locality of a code up to a constant factor [5].)
  • Movassagh-Ouyang Hamiltonian Codes (Movassagh-Ouyang codes stem from a prescription that converts an arbitrary classical code into a quantum code.)

Cousin(s)
  • Binary linear code (Construction uses two related binary linear codes \(C_X\) and \(C_Z\).)
  • Weakly self-dual linear code (CSS codes for which \(C_X=C_Z \equiv C\) are called weakly self-dual since \(C^{\perp} \subseteq C\). The stabilizer group of such codes is invariant under the Hadamard gate exchanging \(X\) and \(Z\).)
  • Group GKP code (For an \(n\)-qubit CSS code, the nested group construction \(C_1^\perp \subseteq C_2 \subset \mathbb{Z}_2^{\times n}\).)
  • \(\mathbb{Z}_q\)-qudit CSS code (Extension of CSS codes to modular-integer qudits.)
  • Galois-qudit CSS code (Extension of qubit CSS codes to Galois qudits.)
  • Majorana stabilizer code (When constructing a Majorana stabilizer code from a weakly self-dual classical code with an odd number of bits and generator matrix \(G\), a more complex procedure must be applied to ensure that the fermion code has an even number of Majorana zero modes, and thus a physical Hilbert space [5][6]. Rather than taking \(G\) to be the stabilizer matrix as in the even case, we take \(G\oplus G\). This is a concatenation of classical codes as in the CSS construction and it yields a mapping \([2N-1,k,d]\rightarrow [[2N-1,2N-1-k,d^\perp]]_f\). This procedure may be further generalized by concatenating two different weakly self-dual classical codes with an odd number of bits, as is often done in the CSS construction.)
