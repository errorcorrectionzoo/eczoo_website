Calderbank-Shor-Steane (CSS) stabilizer code

Name
Calderbank-Shor-Steane (CSS) stabilizer code[1][2][3]
Description

An \([[n,k,d]]\) stabilizer code admitting a set of stabilizer generators that are either \(Z\)-type or \(X\)-type Pauli strings. The parity check matrix is of the form \begin{align} H=\begin{pmatrix}0 & H_{Z}\\ H_{X} & 0 \end{pmatrix}~. \label{eq:parity} \end{align}

Encoding is based on two binary linear codes, an \([n,k_X,d_X]\) code \(C_X\) and \([n,k_Z,d_Z]\) code \(C_Z\) with \(C_X^\perp \subseteq C_Z\), such that \(k=k_X+k_Z-n\) and \(d\geq\min\{d_X,d_Z\}\). The \(H_X\) (\(H_Z\)) block of \(H\) \eqref{eq:parity} is associated with the code \(C_X\) (\(C_Z\)), and the requirement \(C_X^\perp \subseteq C_Z\) guarantees that the \(X\)-stabilizers of \(C_X\) commute with the \(Z\)-stabilizers of \(C_Z\).

A CSS code has stabilizer weight \(w\) if the highest weight of any stabilizer generator is \(w\), i.e., any row and column of \(H_X\) and \(H_Z\) has weight at most \(w\). In the context of comparing weight as well as of noise models biased toward \(X\)- or \(Z\)-type errors, an extended notation for CSS codes is \([[n,k,(d_X,d_Z),w]]\).

There exists a many-to-one mapping from size three chain complexes to CSS codes [4][5][6][7] that allows one to extract code properties from topological features of the corresponding chain complexes. A chain complex of size three is given by binary vector spaces \(C_2\), \(C_1\), \(C_0\) and boundary operators (read: matrices) \(\partial_2\), \(\partial_1\) that satisfy \(\partial_1 \circ \partial_2 = 0\). We can construct a CSS code by associating a physical qubit to every basis element of \(C_1\) and associating vectors of \(Im(\partial_1^T)\) and \(Im(\partial_2)\) with the symplectic representation of stabilizers \(C_x\) and \(C_z\), respectively. Then the normalizer of the stabilizers \(N(C_x)\) and \(N(C_z)\) are \(Ker(\partial_2^T)\) and \(Ker(\partial_1)\), respectively.

Usually, the chain complex used in the above construction comes from the chain complex associated with a cellulation of a manifold. When the manifold is a two-dimensional surface, its entire chain is used. For larger dimensions, one can use the three largest non-trivial vector spaces in its chain.

Codes constructed in the above manner are sometimes called homological codes, but they are equivalent to CSS codes. The reverse mapping is as follows. Given a CSS code with parity check matrices \(H_x\) and \(H_y\), let both the boundary operators be \(\partial = H_z^TUH_x\) for an arbitrary invertible matrix \(U\). Then the code derived from a chain with this boundary operator will have stabilizers \(C_x\) and \(C_z\) be \(Im(\partial^T)\) and \(Im(\partial)\), respectivley.

Protection

Detects errors on \(d-1\) qubits, corrects errors on \(\left\lfloor (d-1)/2 \right\rfloor\) qubits.

Pre the homological-to-CSS mapping, the number of encoded logical qubits is equal to the dimension of the first \(\mathbb{Z}_2\)-homology of the chain complex, \(H_1(\partial, \mathbb{Z}_2) = \frac{Ker(\partial_1)}{Im(\partial_2)}\). The distance of the CSS code is equal to the minimum of the combinatorial (\(d-1\))-systole of the cellulated \(d\)-dimensional manifold and its dual. For surfaces, there is an upper bound of \(d \leq O(\sqrt(n))\) on the distance, therefore codes coming from higher-dimensional spaces are studied for larger distance.

Encoder
Stabilizer measurement [8].
Notes
Original requirement of \(C_X^\perp \subset C_Z\) [1] has been relaxed to absorb hypergraph product codes.

Cousin(s)
  • Binary linear code (Construction uses two related binary linear codes \(C_X\) and \(C_Z\).)
  • Dual linear code (CSS codes for which \(C_X=C_Z \equiv C\) are called weakly self-dual since \(C^{\perp} \subseteq C\). The stabilizer group of such codes is invariant under the Hadamard gate exchanging \(X\) and \(Z\).)
  • Graph homology code (CSS codes can also be constructed using homology techniques but for manifolds of dimension two or greater.)
  • Group GKP code (For an \(n\)-qubit CSS code, the nested group construction \(C_1^\perp \subseteq C_2 \subset \mathbb{Z}_2^{\times n}\).)
  • \(\mathbb{Z}_q\)-qudit CSS code (Extension of CSS codes to modular-integer qudits.)
  • Binary quantum Goppa Code (Goppa codes can be constructed using CSS codes over hyperelliptic curves [10]. The construction provided here is Matsumoto's construction.)
  • Galois-qudit CSS code (Extension of qubit CSS codes to Galois qudits.)
  • Majorana stabilizer code (When constructing a Majorana stabilizer code from a weakly self-dual classical code with an odd number of bits and generator matrix \(G\), a more complex procedure must be applied to ensure that the fermion code has an even number of Majorana zero modes, and thus a physical Hilbert space [9][11]. Rather than taking \(G\) to be the stabilizer matrix as in the even case, we take \(G\oplus G\). This is a concatenation of classical codes as in the CSS construction and it yields a mapping \([2N-1,k,d]\rightarrow [[2N-1,2N-1-k,d^\perp]]_f\). This procedure may be further generalized by concatenating two different weakly self-dual classical codes with an odd number of bits, as is often done in the CSS construction.)
