Dual additive code

Description

For any \(q\)-ary additive code \(C\), the dual additive code is \begin{align} C^\perp = \{ y\in GF(q)^{\times n} ~|~ x \star y=0 \forall x\in C\}, \end{align} where the trace inner product is \(x\star y = \sum_{i=1}^n \text{tr}(x_i y_i)\) for coordinates \(x_i,y_i\), and the trace maps elements of the field \(GF(q)\) with \(q=p^m\) to elements of \(GF(p)\) as \begin{align} \text{tr}(\gamma)=\sum_{k=0}^{m-1}\gamma^{p^{k}}~. \end{align}

A code that contains its dual, \(C^\perp \subseteq C\), is called weakly self-dual additive or self-orthogonal additive. A code that is equal to its dual, \(C^\perp = C\), is called self-dual additive.

Cousins

  • Dual linear code — The difference between the definitions of dual linear and dual additive codes is in the inner product. Self-dual linear codes are also self-dual additive codes.
  • Stabilizer code over \(GF(4)\) — Let \(\phi\) be a bijection from a linear binary subspace to \(GF(4)^n\). Let \(C\) be a self-orthogonal additive subcode over \(GF(4)\), containing \(2^{n-k}\) vectors, such that there are no vectors of weight less than \(d\) in \(C^{\perp}\setminus C\). Then, any eigenspace of the inverse map \(\phi^{-1}(C)\) is an \([[n, k, d]]\) stabilizer code over \(GF(4)\).
  • Stabilizer code over \(GF(q^2)\) — Self-orthogonal additive codes are used in this stabilizer code construction.
