Projective code

Description

Linear binary or \(q\)-ary \([n,k,d]\) code such that columns of its generator matrix \(G\) does not contain any repeated columns or the zero column. That way, each column corresponds to a distinct point in the projective space \(PG(k-1,q)\). If the columns are linearly independent, then the codewords are collectively called an information set.

Recall that a linear code encodes a message \(h\) into a codeword \(c = hG\). The \(i\)th coordinate of a codeword is given the dot product \(h \cdot G_{i}\) with \(G_{i}\) being the \(i\)th column of the generator matrix. The zero-coordinate condition \(h \cdot x = 0\) defines a hyperplane of points \(x\) with normal vector \(h\). Therefore, the Hamming weight of the corresponding codeword is the number of points \(G_i\) not contained in said hyperplane.

Notes

Parent

  • Linear code — Columns of the generator matrix of a projective linear code correspond to distinct points in projective space. In general, linear codes may admit (a multiset of ) non-distinct points.

Children

  • Evaluation AG code — An AG-code generator matrix has columns that form a subset of points on a curve in projective space.
  • Maximum distance separable (MDS) code — An \(n\)-arc in the projective space \(PG_{n-k-1}(q)\) defines an MDS code [1].
  • Simplex code

Cousin

  • Weight-two code — There is a 1-1 correpondence between projective and weight-two codes [2][3].
