Low-density parity-check (LDPC) code[1]

Description

Also known as Gallager codes. Family of \([n,k,d]\) binary or \(q\)-ary linear codes for which the number of nonzero entries in each row and column of the parity check matrix are both bounded by a constant as \(n\to\infty\). In other words, the parity-check matrix of an LDPC code is sparse.

A parity check is performed by taking the inner product of a row of the parity-check matrix with a codeword that has been affected by a noise channel. A parity check yields either zero (no error) or one (error) for binary codes, while yielding zero (no error) or a nonzero field element (error) for \(q\)-ary codes. Despite the fact that there is more than one nonzero outcome, \(q\)-ary linear codes with sparse parity-check matrices are also called LDPC codes.

Rate

Achieve capacity on the binary symmetric channel under maximum-likelihood decoding [1][2]. Some LDPC codes achieve capacity for smaller block lengths under belief-propagation decoding [3]. Random LDPC codes achieve list-decoding capacity [4].

Decoding

Belief propagation using min-sum algorithm [5].Linear programming [6].

Realizations

5G NR cellular communication for the traffic channel [7].WiMAX (IEEE 802.16e) [8].Satellite transmission of digital television [9].

Notes

See database [10] for explicit representatives of several classes of LDPC codes, including \(q\)-ary, WiMAX, multi-edge, and spatially-coupled.See Ref. [11] for a review of LDPC codes circa 2005.Codes have been benchmarked using AFF3CT toolbox [12].
