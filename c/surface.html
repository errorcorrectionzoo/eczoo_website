Kitaev surface code

Name
Kitaev surface code[1][2][3]
Description

A family of CSS stabilizer codes whose generators are few-body \(X\)-type and \(Z\)-type Pauli strings associated to the stars and plaquettes, respectively, of a cellulation of a two-dimensional surface (with a qubit located at each edge of the cellulation). Toric code often either refers to the construction on the two-dimensional torus or is an alternative name for the general construction. The construction on surfaces with boundaries is often called the planar code [4].

The original construction can be naturally extended to arbitrary dimensional manifolds [5]. Given a cellulation, qubits are put on \(i\)-dimensional faces, \(X\)-type stabilizers are associated with \((i-1)\)-faces, while \(Z\)-type stabilizers are associated with \(i+1\)-faces.

Protection
Toric code on an \(L\times L\) torus is a \([[2L^2,2,L]]\) CSS code, and there exists a planar code with \([[L^2,1,L]]\) [6]. More generally, the code distance is related to the homology of the cellulation [7].
Rate
Rate depends on the underlying cellulation and manifold. For general 2D manifolds, \(kd^2\leq c(\log k)^2 n\) for some constant \(c\) [8], meaning that surface codes with finite rate can only achieve an asymptotic minimum distance that is logarithmic in \(n\).
Encoder
Unitary [9][10][11].Dissipative [12].Stabilizer measurement-based [13].
Transversal Gates
Pauli gates on torus geometry.
Gates
Clifford gates can be implemented via lattice surgery [6][14] or braiding defects [15][16][17][18].Non-Clifford gates require magic state distillation [19] or just-in-time decoding [20].
Decoder
Minimum weight perfect-matching [7][21], union-find [22], renormalization group [23][24], tensor network [25], Markov chain Monte Carlo [26], cellular automaton [27], machine learning [28][29][30].
Threshold
\(10.9\%\) with independent \(X/Z\) errors for square tiling [31].\(18.9\%\) with depolarizing noise for square tiling [32].\(50\%\) with loss errors for square tiling [33].\(3.3\%\) with phenomenological noise for square tiling [34].\(0.5-1.1\%\) for various error models [35].
Realizations
Distance-two surface codes have been implemented by Andersen et al. [36], Erhard et al. [37], and Google Quantum AI [38].Signatures of corresponding topological phase of matter detected in superconducting circuits [39] and two-dimensional arrays of Rydberg atoms [40].Distance-three surface code implemented at ETH Zurich [41] and on the Zuchongzhi 2.1 superconducting quantum processor [42].
Notes
2D and 3D surface code visualization tool.Toric code introductions from error-correction perspective by J. Haah and condensed-matter perspective by M. Levin and C. Nayak.
Parent(s)
  • Calderbank-Shor-Steane (CSS) stabilizer code (Plaquette and star operators are stabilizer generators.)
  • Topological code (When treated as ground states of the code Hamiltonian, the code states realize \(\mathbb{Z}_2\) topological order, a topological phase of matter that also exists in \(\mathbb{Z}_2\) lattice gauge theory [43].)
Parent of
  • XZZX surface code (\(XZZX\) code is obtained from the surface code by Hadamard gates.)
  • Projective-plane surface code (Surface code on the real projective plane.)
  • Guth-Lubotzky hyperbolic surface code (The Guth-Lubotzky code is an extension of the surface code to the 4D hyperbolic setting [44].)
  • Two-dimensional hyperbolic surface code (Surface code on a two-dimensional hyperbolic surface.)
Cousin(s)
  • Quantum low-density parity-check (QLDPC) code (Stabilizer generator weights and qubit degrees are given by the properties of the cellulation, and most cellulations yield QLDPC codes.)
  • Hypergraph product code (Planar (toric) code obtained from hypergraph product of two repetition (cyclic) codes.)
  • Color code (Color code is equivalent to surface code in several ways [45]. For example, the color code on a \(D\)-dimensional closed manifold is equivalent to multiple decoupled copies of the \(D\)-dimensional surface code.)
  • Matching code (Matching code realizes the same topological order as the surface code.)
  • Double-semion code (There is a logical basis for the toric and double-semion codes where each codeword is a superposition of states corresponding to all noncontractible loops of a particular homotopy type. The superposition is equal for the toric code, whereas some loops appear with a \(-1\) coefficient for the double semion.)
  • Honeycomb code (Measurement of each check operator involves two qubits and projects the state of the two qubits to a two-dimensional subspace, which we regard as an effective qubit. These effective qubits form a surface code on a hexagonal superlattice. Electric and magnetic operators on the embedded surface code correspond to outer logical operators of the Floquet code. In fact, outer logical operators transition back and forth from magnetic to electric surface code operators under the measurement dynamics.)
  • \([[4,2,2]]\) CSS code (\([[4,2,2]]\) code is the smallest toric code.)
