Topological code

Name
Topological code
Description
Stub.
Parent(s)
  • Hamiltonian-based code (Codespace is either the ground-state or low-energy subspace of a geometrically local Hamiltonian admitting a topological phase.)
Parent of
  • Color code (Defined on a lattice with geometrically local stabilizer generators [1].)
  • Matching code (Matching codes were inspired by the Kitaev honeycomb model [2], which realizes \(\mathbb{Z}_2\) topological order.)
  • Double-semion code (When treated as ground states of the code Hamiltonian, the code states realize double-semion topological order, a topological phase of matter that also exists in twisted \(\mathbb{Z}_2\) gauge theory [3].)
  • Kitaev surface code (When treated as ground states of the code Hamiltonian, the code states realize \(\mathbb{Z}_2\) topological order, a topological phase of matter that also exists in \(\mathbb{Z}_2\) lattice gauge theory [4].)
Cousin(s)
  • Quantum low-density parity-check (QLDPC) code (Stabilizer-based topological codes (e.g., the surface code) on appropriate tesselations are QLDPC.)
  • Local Haar-random circuit code (Local Haar-random codewords, like topological codewords, are locally indistinguishable [5].)
  • Monitored Random Circuits (Topological order can be generated in 2D monitored random circuits [6])
  • Fracton code
  • Fusion Based Quantum Computing (Arbitrary topological codes can be created using FBQC, as can topological features such as defects and boundaries, by modifying fusion measurements or adding single qubit measurements.)
